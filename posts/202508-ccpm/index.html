📚 Claude Code PM - 项目管理系统

🎯 快速开始工作流程

  1. /pm:prd-new <name> - 创建新的 PRD
  2. /pm:prd-parse <name> - 将 PRD 转换为史诗任务
  3. /pm:epic-decompose <name> - 分解为任务
  4. /pm:epic-sync <name> - 推送到 GitHub
  5. /pm:epic-start <name> - 开始并行执行

📄 PRD 命令

  • /pm:prd-new <name> - 为新产品需求启动头脑风暴
  • /pm:prd-parse <name> - 将 PRD 转换为实现史诗任务
  • /pm:prd-list - 列出所有 PRD
  • /pm:prd-edit <name> - 编辑现有 PRD
  • /pm:prd-status - 显示 PRD 实现状态

📚 史诗任务命令

  • /pm:epic-decompose <name> - 将史诗任务分解为任务文件
  • /pm:epic-sync <name> - 将史诗任务和任务推送到 GitHub
  • /pm:epic-oneshot <name> - 一步完成分解和同步
  • /pm:epic-list - 列出所有史诗任务
  • /pm:epic-show <name> - 显示史诗任务及其任务
  • /pm:epic-status [name] - 显示史诗任务进度
  • /pm:epic-close <name> - 标记史诗任务为完成
  • /pm:epic-edit <name> - 编辑史诗任务详情
  • /pm:epic-refresh <name> - 从任务更新史诗任务进度
  • /pm:epic-start <name> - 启动并行代理执行

📝 Issue 命令

  • /pm:issue-show <num> - 显示 issue 和子 issues
  • /pm:issue-status <num> - 检查 issue 状态
  • /pm:issue-start <num> - 开始工作与专门代理
  • /pm:issue-sync <num> - 将更新推送到 GitHub
  • /pm:issue-close <num> - 标记 issue 为完成
  • /pm:issue-reopen <num> - 重新打开已关闭的 issue
  • /pm:issue-edit <num> - 编辑 issue 详情
  • /pm:issue-analyze <num> - 分析并行工作流

🔄 工作流程命令

  • /pm:next - 显示下一个优先任务
  • /pm:status - 整体项目仪表板
  • /pm:standup - 每日站会报告
  • /pm:blocked - 显示被阻塞的任务
  • /pm:in-progress - 列出进行中的工作

🔗 同步命令

  • /pm:sync - 与 GitHub 完全双向同步
  • /pm:import <issue> - 导入现有的 GitHub issues

🔧 维护命令

  • /pm:validate - 检查系统完整性
  • /pm:clean - 归档已完成的工作
  • /pm:search <query> - 搜索所有内容

⚙️ 设置命令

  • /pm:init - 安装依赖并配置 GitHub
  • /pm:help - 显示此帮助消息

💡 提示

  • 使用 /pm:next 查找可用工作
  • 运行 /pm:status 获取快速概览
  • 史诗任务工作流程: prd-new → prd-parse → epic-decompose → epic-sync
  • 查看 README.md 获取完整文档

commands

context

  • context:create: 此命令通过分析当前项目状态并建立全面的基线文档，在 .claude/context/ 中创建初始项目上下文文档。
  • context:prime: 加载上下文文档
  • context:update: 更新上下文文档

pm

  • pm:blocked: 扫描项目目录，列出所有状态为“开放”且因依赖于其他尚未完成任务而被阻塞的任务
  • pm:clean: 清理已完成的工作并归档旧的史诗任务
  • pm:epic-close: 当所有任务完成时将史诗任务标记为完成
  • pm:epic-decompose: 将史诗任务分解为具体、可操作的任务。
  • pm:epic-edit: 创建后编辑史诗任务详情。
  • pm:epic-list: 读取项目史诗任务（epics）文件，并按规划中、进行中和已完成三种状态分类显示其详细信息及总体摘要
  • pm:epic-merge: 将已完成的史诗任务从工作树合并回主分支。
  • pm:epic-oneshot: 将 Epic 分解为任务并同步到 GitHub 的一键操作。
    • 该命令会运行epic-decompose, epic-sync
  • pm:epic-refresh: 根据任务状态更新史诗任务进度。
  • pm:epic-show: 显示指定史诗任务的详细信息，包括其元数据、各项子任务的完成状态和总体进度统计，并提供下一步可能的操作建议。
  • pm:epic-start-worktree: 在共享工作树中启动并行代理来处理史诗任务。
  • pm:epic-start: 在共享分支中启动并行代理来处理史诗任务。
  • pm:epic-status: 分析一个指定的“史诗任务”目录下的Markdown文件，计算并显示出该任务的总体进度、任务完成情况统计以及相关的GitHub链接。
  • pm:epic-sync: 将 epic 和任务推送到 GitHub issue。
